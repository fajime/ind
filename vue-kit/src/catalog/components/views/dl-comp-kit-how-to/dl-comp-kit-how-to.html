Ayuda

Para poder añadir un componente a este kit de componentes es necesario cumplir los siguientes requisitos:
  • El componente tiene que ser autocontenido. No se permiten componentes que no sean íntegros.
  • Si el componente contiene otros componentes hijos, estos deben estar incluidos también en este mismo kit.
  • Tiene que tener estilos personalizables tal y como a continuación se explica
  • No se permite lógica de aplicación dentro de un componente.
  • Todos los componentes tienen que tener sus test asociados y no pueden ser falseados
Rellena los siguientes campos con el nombre del componente que deseas generar y con el tipo de componente y podrás ver los pasos :
Para generar un componente es necesario seguir una estructura de carpetas y una serie de reglas que a continuación se detallan:
  1. Posicionarse en la carpeta /src/components/{{componentType}}s
  2. Crear una carpeta con el nombre dl-ui-{{componentNameNormalized}}
  3. Dentro de la carpeta del componente generar la siguiente estructura:
    Carpeta contenedora del componente dl-ui-{{componentNameNormalized}}
    Fichero punto de entrada del componente index.vue
    Tempalte HTML del componente dl-ui-{{componentNameNormalized}}.html
    Parte JS del componente dl-ui-{{componentNameNormalized}}.js
    Punto de entrada de generación de estilos dl-ui-{{componentNameNormalized}}.scss
    Carpeta contenedora de mixins de estilos mixins
    Mixin de generación de estilos dinámicos _dl-ui-{{componentNameNormalized}}.scss
  4. El fichero index.vue contiene única y exclusivamente una referencia a los ficheros dl-ui-{{componentNameNormalized}}.html, dl-ui-{{componentNameNormalized}}.js y dl-ui-{{componentNameNormalized}}.css.
  5. El fichero dl-ui-{{componentNameNormalized}}.html contiene el template HTML del componente. El elemento raíz del template no debe llevar ninguna clase asociada puesto que será cuando se utilice el componente cuando se asigne, de esta forma se posibilita su personalización
  6. El fichero dl-ui-{{componentNameNormalized}}.js contiene la declaración del componente. Es obligatorio declara el campo name del mismo con el valor dl-ui-{{componentNameNormalized}}.
  7. El fichero dl-ui-{{componentNameNormalized}}.scss contiene al menos una clase denominada .dl-ui-{{componentNameNormalized}}. Dentro de esta clase se realizará la llamada al mixin incluido en el fichero mixins/_dl-ui-{{componentNameNormalized}}.css pasándole un mapa de SCSS vacío para forzar la generación de los estilos con el diseño estandar.
  8. El fichero mixins/_dl-ui-{{componentNameNormalized}}.scss contiene la declaración de un mixin, el cual recibe un parámetro mapa clave-valor denominado $config. El nombre del mixin deberá ser {{componentNameNormalized}}-style-config. Dentro de este mixin se declaran las clases y propiedades CSS que necesita el componente para visualizarse. Todo elemento que permite su personalización deberá ser configurado de la siguiente forma propiedad-css: getMapValue($config, 'nombre-variable-mapa', valor-por-defecto);, como por ejemplo padding: getMapValue($config, 'title::padding', 1rem);.
  9. El nombre de la variable que contiene el mixin se compone de un nombre descriptivo, dos signos dos puntos consecutivos y la propiedad CSS que modifica. Ejemplos title::height, bar::background-color o header::font-size.
Para generar la demo de un componente es neceasrio seguir una estructura de carpetas y una serie de reglas que a continuación se detallan:
  1. Posicionarse en la carpeta /src/components/{{componentType}}s/dl-ui-{{componentNameNormalized}}
  2. Crear una carpeta con el nombre __DEMO
  3. Dentro de la carpeta del componente generar la siguiente estructura:
    Carpeta contenedora de la demo del componente __DEMO
    Fichero punto de entrada de la demo del componente index.vue
    Tempalte HTML de la demo del componente demo__dl-ui-{{componentNameNormalized}}.html
    Parte JS de la demo del componente demo__dl-ui-{{componentNameNormalized}}.js
    Estilos de la demo del componente demo__dl-ui-{{componentNameNormalized}}.scss
    Carpeta contenedora de ficheros de metadatos del componente data
    Metadatos relacionados con la descrición del componente demo__dl-ui-{{componentNameNormalized}}.desc.js
    Metadatos relacionados con los eventos del componente demo__dl-ui-{{componentNameNormalized}}.events.js
    Metadatos relacionados con las propiedades del componente demo__dl-ui-{{componentNameNormalized}}.properties.js
    Metadatos relacionados con las propiedades SCSS dinámicas demo__dl-ui-{{componentNameNormalized}}.scss.js
    Metadatos relacionados con los slots del componente demo__dl-ui-{{componentNameNormalized}}.slots.js
    Metadatos relacionados con datos de notas del componente demo__dl-ui-{{componentNameNormalized}}.notes.js
  4. El fichero index.vue contiene única y exclusivamente una referencia a los ficheros demo__dl-ui-{{componentNameNormalized}}.html, demo__dl-ui-{{componentNameNormalized}}.js y demo__dl-ui-{{componentNameNormalized}}.css.
